OSPCAT

SYNOPSIS

ospcat [ -vC ] [ -bbctf ] [ -ferror_file ] [ -ccatalog_sysid ] [ -Ddirectory ] [ -Ppublic_id ] [ -Ssystem_id ] [ -pparameter_entity ] [ -ddoctype ] [ -llinktype ] [ -eentity ] [ -nnotation ] [ -sdoctype ]

DESCRIPTION

Ospcat (SGML print catalog) prints effective system identifiers found by sp in the catalogs on the standard output. It doesn't check that the system identifiers correspond to actual files. The return value is 0 if the last system identifier was successfully created and 1 otherwise.

The following options are available:

-C
-bbctf
-ferror_file
-ccatalog_sysid
-Ddirectory
These options have the same effect as in onsgmls.
-v
Print the version number.
-Ppublic_id
Print the effective system identifier for the public identifier public_id.
-Ssystem_id
Print the effective system identifier for the system identifier system_id.
-pparameter_entity
Print the effective system identifier for the parameter entity with the name parameter_entity.
-ddoctype
Print the effective system identifier for the declaration of the document type with the name doctype.
-llinktype
Print the effective system identifier for the declaration of the link type with the name linktype.
-eentity
Print the effective system identifier for the general entity with the name entity.
-nnotation
Print the effective system identifier for the notation with the name notation.
-sdoctype
Print the effective system identifier for the SGML declaration associated with the document type doctype. Until sp implements the DTDDECL catalog entry, this will just return the value of the SGMLDECL entry.
