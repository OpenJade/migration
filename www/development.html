OpenJade Development

OpenJade Releases

OpenJade releases are tagged jade_x_y or jade_x_y_z; for example OpenJade 1.2.1 is tagged jade_1_2_1.

The Bleeding Edge

OpenJade development is proceeding on two parallel branches: the experimental branch, which will lead to OpenJade 1.3, and the stable branch, which will lead to 1.2.2. As can be expected, the stable branch contains mostly bugfixes and simple additions, while the experimental branch contains new, untested features.

Checking out OpenJade will give you the experimental branch. To get the stable branch checkout (or update to) jade_1_2_1_branch. For example,

cvs -z9 checkout jade   # get experimental branch
cvs -z9 update -r jade_1_2_1_branch jade  # switch to stable branch

Logs of development activity are kept in the ChangeLog and the NEWS files. The former is generated automatically from the cvs commit logs, while the latter is manually updated by the maintainers, and is more user-level.

OpenJade Internals Documentation

Some documentation on OpenJade's internals has been written by Matthias Clasen, one of the OpenJade maintainers.

Submitting Patches

Preferrably, start with a clean Jade release and apply your changes to it. Then generate a patch with the command

cvs -z9 diff -r jade_1_2_1 > patch

Pack the patch using the tools appropriate for your platform (Zip for windows or tar -z for *ix) and send it to one of the maintainers, with a note describing the patch and the base version.
